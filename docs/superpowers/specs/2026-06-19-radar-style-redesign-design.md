# 雷达今日变化 · 样式重设计

**日期**: 2026-06-19  
**组件**: `src/components/DailyRead.tsx`（雷达区 section）  
**状态**: 已批准，待实现

---

## 问题

现有雷达区所有事件类型（首现 / 升温 / 降温 / 位置前移 / 口径翻转）渲染为完全相同的样式：同色（`#c99`）、同字号（12px）、Unicode 图标字符辨识度低。读者无法在 3 秒内区分今日信号的类型与量级，阅读体验差。

---

## 设计目标

- 让读者在 **3 秒内** 扫描出今日有什么显著信号、是什么类型、量级如何
- 词条（target）是视觉主角，类型是快速锚点，强度是辅助信息
- 与页面整体深色 editorial 风格（`#08080e` 背景、`#ECE7DD` 主文字）保持一致

---

## 设计方案：固定图标列 + 色调底色（E-2）

### 整体结构

雷达列表整体包裹在一个 `border: 1px solid #1b1b26; border-radius: 6px` 容器内。每条雷达事件为一行，行间以 `1px solid #1b1b26` 分割。

每行分两列：

```
┌──────────┬──────────────────────────────────────┐
│  图标列  │  内容列                               │
│  56px    │  flex: 1                              │
│  色调底色│  词条 (15px bold) + 强度条 or 类型名  │
└──────────┴──────────────────────────────────────┘
```

### 图标列（56px 固定宽）

- 大图标：`font-size: 20px`，类型专属颜色
- 短类型标：`font-size: 8px`，`letter-spacing: 0.5px`，同色
- 背景：每种类型有独立的极淡色调底色
- 右侧分割线：`1px solid` 色调略深于底色

### 内容列（`padding: 11px 16px`）

- **词条**：`font-size: 15px; font-weight: 700; color: #ECE7DD`（始终主角）
- **有强度时**：词条下方显示 `72px` 进度条 + 数字（颜色与类型一致）
  - 进度条满值 = 当日 radar 列表中的最大 magnitude
  - track 背景色与图标列底色对应
- **无强度时**：词条下方显示完整类型名，`font-size: 10px; color: #4a4a5a`

### 色彩系统（5 种类型）

| 类型 key | 短标 | 图标 | 前景色 | 图标列底色 | 分割线色 |
|----------|------|------|--------|-----------|---------|
| `new_tifa` | 首现 | ✦ | `#f5c842` | `#141108` | `#1e1a08` |
| `drumbeat_up` | 升温 | ▲ | `#4ade80` | `#0b1a10` | `#0e2018` |
| `drumbeat_down` | 降温 | ▼ | `#f87171` | `#180b0b` | `#200e0e` |
| `order_jump` | 前移 | ↑ | `#38bdf8` | `#081520` | `#0a1e2e` |
| `flip` | 翻转 | ⇅ | `#c084fc` | `#130a1c` | `#180e22` |

### 强度进度条 track 背景色

| 类型 | track 色 |
|------|---------|
| `new_tifa` | —（无强度） |
| `drumbeat_up` | `#1a2e1a` |
| `drumbeat_down` | `#2e1a1a` |
| `order_jump` | `#0a1e2e` |
| `flip` | — （通常无强度） |

## 边界情况

| 场景 | 处理 |
|------|------|
| `magnitude` 为 `null` | 不渲染进度条，显示完整类型名 |
| `radar` 数组为空 | 维持现有"今日无显著雷达变化"空状态文字 |
| 未知 `type` 字符串 | 回退到 `radarIcon()` / `radarLabel()` 的 passthrough 逻辑，使用默认灰色（`#8a8a98`）和纯暗底色 |
| 条目数 > 12 | 数据层已限制 `limit = 12`，无需 UI 截断 |
